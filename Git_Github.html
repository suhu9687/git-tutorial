一. Git

1. Git基础

1.1 Git基础配置

在使用 git 前，需要告诉 git 你是谁，在向 git 仓库中提交时需要用到。

  1. 配置提交人姓名：git config --global user.name "用户名"
  2. 配置提交人邮箱 : git config --global user.email "邮箱地址"
  3. 查看git配置信息　git config --list

注意:

  1. 如果要对配置信息进行修改,重复上述命令即可.
  2. 配置只需要执行一次.

1.2 提交步骤

  1. git init 初始化git仓库
  2. git status 查看文件状态
  3. git add 文件名 添加文件至git缓冲区
  4. git commit -m 提交信息 向仓库中提交代码
  5. git log 查看提交记录

1.3 恢复记录

git rest --hard commit ID 将git仓库中指定的更新记录恢复出来,并且覆盖暂存区和工作目录.

commit ID 获取 : git log 或 git log --pretty=oneline

注意：

回到过去之后，要想再回到之前最新的版本的时候，则需要使用指令去查看历史操作，以得到最新的commit id。

指令：git reflog

1.4 撤销

  • 用暂存区中的文件覆盖工作目录中的文件 : git chechout 文件
  • 将文件从暂存区中删除 : git rm --cached 文件
  • 将git仓库中指定的更新记录恢复出来,并且覆盖暂存区各工作目录 :git rest --hard commit ID

2. Git 进阶

２.１分支

git每次提交后都会有记录，Git把它们串成时间线，形成类似于时间轴的东西，这个时间轴就是一个分支，我们称之为master分支。在开发的时候往往是团队协作，多人进行开发，因此光有一个分支是无法满足多人同时开发的需求的，且在分支上工作并不影响其他分支的正常使用，会更加安全，Git鼓励开发者使用分支去完成一些开发任务.

2.2 分支的相关操作

  1. 查看分支：git branch
  2. 创建分支：git branch 分支名
  3. 切换分支：git checkout 分支名
  4. 删除分支：git branch -d 分支名
  5. 合并分支：git merge 被合并的分支名
  6. 创建新分支，并切换到分支：git checkout -b 分支名

2.3 暂时保存更改

在gitk中 , 可以暂时提取分支上所有的改动并存储 , 让开发人员得到一个干净的工作副本 , 临时转向其它的工作使用场景 : 分支临时切换

  • 存储临时改动 : git stash
  • 恢复改动 : gitstash pop

二．ＧitＨub

当你需要和你的团队共享数据时 , 设置一个远程仓库是很有必要的 , 你可以把它想成一个 " 文件管理服务器 ", 利用这个服务器可以与开发团队和其它成员进行数据交换 .

1.1 仓库创建

  1. Https方式

    • 推送至仓库 : git push github仓库地址 分支名
    • 给仓库起别名方便以后推送 : git remote add origin(别名) 仓库地址
    • 以后推送就可以用 : git push origin master
    • 加参数提交以后可以直接使用git push : git push -u origin master

  1. SSH协议

    • 生成密钥 : ssh-keygen
    • 密钥存储目录 : C:\Users\用户\.ssh
    • 公钥名称 : id_rsa.pub
    • 私钥名称 :id_rsa

1.2 克隆仓库

  • 将存储库克隆到目录中 : git clone 仓库地址
  • 拉取仓库代码 : git pull 远程仓库地址 分支名称
  • 从另一个存储库下载对象和引用 : git fetch 仓库地址 分支名称
